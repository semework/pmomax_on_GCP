Overview

PMOMax converts unstructured project inputs (documents, spreadsheets, notes) into a structured Project Initiation Document (PID) and companion planning artifacts: objectives, scope, schedule, risks, governance, budget and exports. It combines a deterministic parser with an interactive Gemini-based AI assistant plus specialized Risk & Compliance agents and a Gantt visualizer/exporter.

Core features

  • Input Parser: Extracts 28 canonical PID fields from Word/PDF/text inputs.
  • AI Assistant (Gemini): Interactive drafting, summarization, gap-filling via the Generative Language API.
  • Risk & Compliance Agents: Automated scans that flag missing mitigations and governance items.
  • Gantt Visualizer: Generate timelines from schedule inputs; export as images/PDF embedded in docs.
  • Export Suite: Word/DOCX, PDF, JSON and ZIP exports (including Gantt visuals).
  • Examples & Templates: Starter PIDs and examples to speed adoption; Create flow supports paste, upload, or drop inputs.

Quick start — local development

Prerequisites: Node.js (16+ recommended), yarn or npm, and optional Python for doc generation.

# Install
npm install

# Run dev server (Vite)
npm run dev

# Build (production)
npm run build

# Serve the build locally (for static preview)
npm run preview

Notes:

  • The single canonical React entrypoint is App.tsx. Compiled artifacts and temporary JS duplicates (e.g., *.js equivalents of *.tsx) are quarantined.
  • If local AI parsing requires Google credentials, set GOOGLE_API_KEY in your environment or use the demo data path.

Backend & API

The server exposes simple endpoints used by the SPA. See server.mjs and openapi.yaml for the full contract.

EndpointPurpose
POST /api/ai/parseSend raw text or file conversion; returns { ok: true, pid: PMOMaxPID }. Deterministic parser expects canonical schema (28 fields).
POST /api/ai/assistantInteractive assistant backed by Gemini — conversation and drafting calls.
POST /api/ai/riskRisk agent analysis for a PID (used proactively to precompute agent outputs).
POST /api/ai/complianceCompliance scanning agent for governance entries.
GET /api/healthBasic health check.

PMOMax PID schema & validation

The canonical schema (28 fields) is enforced by types.ts and lib/validatePMOMaxPID.ts. Parser prompts are defined in lib/parseUpload.ts and must output strict JSON. Use the demo parser via data/demoData.ts for offline testing.

Exports & document generation

Export helpers live under lib/export.ts and client helpers under lib/exportUtils.js. DOCX/PPTX generation scripts are in scripts/ and require python-docx / python-pptx if you regenerate programmatically.

# Example: generate docs from Markdown (requires Python deps)
python3 scripts/generate_docs.py

Deployment notes

  • We support Cloud Run and a GKE option for larger Enterprise tenants. See deploy.sh and deploy-gke.sh.
  • To avoid packaging large or temporary artifacts during deploy, ensure .gcloudignore and .dockerignore include docs/, Office temporary files (~$*), and other generated exports.
  • Recommended Cloud Run deploy (example): 
    gcloud run deploy YOUR_SERVICE_NAME --source . --region YOUR_REGION --platform managed
  • If you build a container image: 
    gcloud builds submit --tag gcr.io/PROJECT/pmomax:TAG .
gcloud run deploy YOUR_SERVICE_NAME --image gcr.io/PROJECT/pmomax:TAG --region YOUR_REGION --platform managed

Troubleshooting

  • gcloud crashes on deployment complaining about missing files: ensure your .gcloudignore excludes Office temp files; remove any ~$* entries under docs/ before deploying.
  • Chrome flicker/layout issues: ensure App.tsx is the entrypoint and there are inline background fallbacks; dev server run will show exact behavior.
  • Large duplicates or archives: Quarantine duplicates to a folder outside the repo (e.g. ../_pmo0001_quarantine/).

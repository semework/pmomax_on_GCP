Executive summary

PMOMax (PID Architect) extracts structured Project Initiation Documents (PIDs) from unstructured inputs and transforms them into board-ready planning artifacts: objectives, scope, schedule, risks, governance, budget and exports. It combines a deterministic parser, a Gemini-based AI assistant, specialized Risk & Compliance agents, a Gantt visualizer with multiple style presets, and an export suite (DOCX/PDF/JSON/ZIP).

What's included

  • Deterministic Input Parser (28-field PMOMax PID schema)
  • AI Assistant powered by Gemini (interactive drafting & summarization)
  • Risk & Compliance Agents — automated scans, proactive precompute, and Governance merging
  • Gantt Visualizer — critical path, multiple styles, exports
  • Exports — DOCX/PPTX/PDF/JSON/ZIP (scripts under scripts/)
  • Dev & Deploy helpers — Vite dev server, deploy.sh, deploy-gke.sh

New / Important functional notes (2026)

  • Risk & Compliance agents: when a PID is generated the client proactively POSTs to /api/ai/risk and /api/ai/compliance to compute agent outputs. Results are merged into the Governance and Risks sections so agent buttons are instant.
  • Precompute strategy: Agents run on PID load/creation; UI shows precomputed results and marks when results are from agents vs. user-entered items.
  • Create input improvements: the Create flow supports paste, upload, or drop — added safe drop handler and fallback prompt seeding for quick project drafts.
  • Single TypeScript entrypoint: `App.tsx` is canonical; compiled `.js` duplicates are quarantined and ignored for builds.

User guide highlights

Start a project

Open the web app and either: Paste text, Upload a Word/PDF/XLSX, or Drop files into the Create panel. The parser extracts the canonical fields into a draft PID. If parsing is unavailable, use the Create flow to seed a project from prompts.

Using the AI Assistant

The Assistant is field-aware and returns answers grounded in the current PID data. Use it to:

  • Draft or rewrite sections (Executive Summary, Objectives)
  • Ask for enumerations ("List missing mitigations") — the assistant quotes actual PID fields
  • Request short board-ready bullets or longer narrative drafts

Risk & Compliance workflow

Two specialized agents analyze the PID:

AgentOutputWhere it's shown
Risk agentList of identified risks, severity, suggested mitigationsRisks panel + included in exported Governance
Compliance agentMissing governance controls, suggested ownersGovernance panel + admin summary

Behavior: Agents are run automatically on PID creation/updates. Agent outputs are merged with existing user-entered items; buttons in the Governance/Risks panels will show precomputed content for low-latency UX.

Exports & sharing

Use the Export menu to generate DOCX, PDF, JSON, or a ZIP bundle containing the PID and Gantt visuals. If you programmatically regenerate Word/PPTX, use the Python scripts in scripts/ (requires python-docx / python-pptx).

Developer guide

Local dev

# Install dependencies
npm install

# Dev server (Vite)
npm run dev

# Build
npm run build

# Preview built site
npm run preview

Notes: If AI parsing needs credentials, set GOOGLE_API_KEY in your environment. Use demo data via data/demoData.ts for offline tests.

API endpoints

EndpointNotes
POST /api/ai/parseAccepts plain text or parsed PDF conversion; returns { ok:true, pid: PMOMaxPID }
POST /api/ai/assistantConversation and drafting calls to Gemini
POST /api/ai/riskReturn risk analysis array; called proactively by client
POST /api/ai/complianceReturn compliance suggestions; called proactively by client
GET /api/healthBasic health check

Schema & validation

The canonical PMOMax PID schema (28 fields) is defined in types.ts and validated by lib/validatePMOMaxPID.ts. Parsers and assistant prompts rely on strict JSON outputs — see lib/parseUpload.ts for the schema prompt used to instruct the LLM.

Deployment (Cloud Run example)

# From repo root (packaged source deploy)
gcloud run deploy YOUR_SERVICE_NAME --source . --region YOUR_REGION --platform managed

# Or build and push image
gcloud builds submit --tag gcr.io/PROJECT/pmomax:TAG .
gcloud run deploy YOUR_SERVICE_NAME --image gcr.io/PROJECT/pmomax:TAG --region YOUR_REGION --platform managed

Ensure .gcloudignore excludes docs/, Office temp files (~$*), and large archives to avoid upload crashes.

Troubleshooting & common fixes

  • gcloud crash about missing temp file: search and remove Office lock files (~$*) under docs/, or ensure they are listed in .gcloudignore/.dockerignore. Use gcloud meta list-files-for-upload --source . to see uploads.
  • Chrome landing flicker: confirm App.tsx is the SPA entry and that inline background fallbacks are present to avoid flash-of-white.
  • Slow agent responses: Risk & Compliance are precomputed on PID load to keep buttons fast; check server `/api/ai/risk` & `/api/ai/compliance` latency and use caching.

Operations & security

  • Store API keys in Secret Manager; example code reads secrets in server.mjs.
  • Use Firestore for PID state; backups and retention should be part of your infra plan.
  • For Enterprise, prefer dedicated projects and SSO (Identity Platform) and consider running the model calls from a VPC/private egress.

PMOMax — PID Architect

Turn unstructured inputs into board-ready Project Initiation Documents (PIDs). Deterministic parsing, project-aware AI, agent analysis, and export-ready outputs.

Explore features Open user guides

Overview

PMOMax ingests documents, spreadsheets and text; parses them into a canonical 28-field PID; runs optional AI-backed enrichment and agent analyses; and exports high-fidelity Word, PDF and data artifacts. The app now includes a proactive agent pipeline: Risk and Compliance agents run automatically on PID load to surface governance items and mitigations.

Key Features

  • 28-field canonical PID — enforced schema, strict JSON export and validator in lib/validatePMOMaxPID.ts.
  • Hybrid parser — local heuristics plus optional Gemini parsing for OCR and fuzzy mappings.
  • Project-aware AI Assistant — answers only from the PID data and quotes fields verbatim when requested.
  • Interactive Gantt — multiple styles, critical-path highlighting, reversed Y-axis, and exportable visuals.
  • Export Suite — DOCX/PDF/JSON/CSV/Markdown with embedded Gantt images and governance tables.
  • Proactive Agents — Risk & Compliance agents precompute analysis on PID load (details below).

Parser & Inputs

Supported inputs: .docx, .pdf, .txt, .xlsx, .md, .csv. Use headings and table semantics for best mapping. Large files can be parsed in chunks to improve reliability.

Assistant Behavior

The AI assistant is constrained to the PID: it will enumerate present sections, quote fields, and indicate missing items. It will not invent facts outside the PID. Two assistant entrypoints are available: a left-panel assistant and a Create-mode assistant (both support expand/collapse and a close (X) control).

Agents — Risk & Compliance

PMOMax ships with two domain-specific agents designed to accelerate governance reviews. These agents run proactively when a PID is loaded or generated and their results are merged into the Governance and Risks views so users can act quickly.

Risk Agent

  • Scans the PID for missing mitigations, high-impact items, and inconsistent severity/likelihood pairs.
  • Generates suggested mitigations and an initial risk rating (probability × impact).
  • Outputs are stored and surfaced in the Risks table and Risk Matrix; clickable items jump to relevant PID sections.

Compliance Agent

  • Checks governance fields, decision-makers, approvals, and required artifacts against a configurable checklist.
  • Flags missing approvals, unclear owner assignments, and potential policy gaps.
  • Designed for rapid triage: results appear in Governance and Compliance sections and are available for export.

Operational Notes

  • Agents run via backend endpoints: POST /api/ai/risk and POST /api/ai/compliance. They can be triggered manually from the UI or precomputed automatically on PID load.
  • Precomputed outputs are cached per PID to make agent buttons fast — the UI merges agent outputs with user-entered items when rendering Risks/Governance.
  • Agent suggestions are recommendations only; users must review and accept changes into the canonical PID.

Guides & User Manual (highlights)

This section summarizes key user workflows and points to downloadables and deeper guides included in the repository.

Quick workflows

  • Load demo — useful for training and acceptance tests.
  • Upload / Drop — upload DOCX/PDF/XLSX/MD/CSV or paste raw text into the parse area.
  • Refine with Assistant — ask for field quotes, reorder tasks, or request mitigation suggestions from the Risk Agent.
  • Export — Word/PDF for governance; JSON/CSV for downstream analytics.

Agent usage in the User Guide

The full user manual explains where to find agent results and how to act on them. Key steps:

  1. Load or generate a PID.
  2. Wait for automatic agent precomputation (UI shows loading state), or click the Risk / Compliance buttons to run on-demand.
  3. Review agent findings in the Risks and Governance panels — suggestions show source evidence and confidence.
  4. Accept, adjust, or dismiss suggestions; accepted items are merged into the PID and become canonical for export.

Parsing behavior & best practices

PMOMax combines deterministic heading/table heuristics with optional Gemini parsing for ambiguous content. If you have many large PDFs, consider splitting or running the parser in stages. The parser attempts to preserve table semantics and maps recognized columns into the canonical PID arrays.

Troubleshooting parsing

  • If sections are missing, check heading styles in the source. Use the AI assistant to move paragraphs into the right section.
  • For scanned PDFs, enable OCR in the environment or use Document AI as part of the intake pipeline.

Exports & Templates

Exports are generated using templates in lib/export.ts. DOCX and PPTX generation scripts live in scripts/. For automated CI-based document generation, install the Python deps listed in requirements.txt (python-docx, python-pptx, Pillow, lxml).
